# ai-pixels 指令规范 v1

## 设计原则

1. **省 token**：数组格式，无 key 名，短类型码
2. **精确**：像素级坐标，明确参数位置
3. **可扩展**：类型码注册制，尾部可选参数，版本号兜底

## 整体结构

```jsonc
{
  "v": 1,                    // 指令版本，向后兼容
  "s": [                     // steps 指令数组
    ["C", 32, 32],           // 每条指令是一个数组
    ["pal", ["#f00", "#0f0"]],
    ["p", 5, 10, 0]
  ]
}
```

项目保存时使用完整 `s` 数组。LLM 交互时只返回 `s` 中除 `C` 之外的部分，前端会按当前对话画布尺寸自动补上 `C`。

## 画布显示设置（前端渲染层，不进指令）

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 网格显示 | 开 | 像素格之间的分隔线 |
| 网格颜色 | `rgba(0,0,0,0.15)` | 浅色，不干扰画面 |
| 网格阈值 | 缩放 < 4x 时自动隐藏 | 缩得太小网格无意义，自动关闭 |
| 背景样式 | 棋盘格 | 透明区域用棋盘格表示 |
| 缩放级别 | 自适应 | 每像素格映射到屏幕像素数 |

网格渲染策略：
- 用独立的 overlay canvas 叠在主画布上方绘制网格线，避免污染像素数据
- 导出 PNG 时不包含网格
- 截图发给 AI 时不包含网格（取主画布的 toDataURL）
- 缩放倍率低于阈值时自动隐藏网格，避免密集线条糊成一片

## 类型码注册表

| 码 | 名称 | 说明 |
|----|------|------|
| `C` | canvas | 画布初始化 |
| `pal` | palette | 调色板定义 |
| `p` | pixel | 单像素 |
| `P` | pixels | 批量像素 |
| `r` | rect | 矩形 |
| `e` | ellipse | 椭圆/圆 |
| `l` | line | 直线 |
| `f` | flood | 填充 |
| `#` | comment | 注释 |

预留扩展码（暂不实现）：

| 码 | 名称 | 说明 |
|----|------|------|
| `t` | text | 像素文字 |
| `m` | mirror | 镜像 |
| `cp` | copy | 区域复制 |
| `sp` | sprite | 精灵图引用 |
| `rr` | rrect | 圆角矩形 |
| `pl` | polyline | 折线 |
| `cl` | clear | 清除区域 |

新增类型只需注册码 + 定义参数序列，不影响已有指令。

## 指令详细定义

### `C` - 画布初始化

```
["C", width, height]
["C", width, height, bg]
```

| 位置 | 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| 1 | width | int | 是 | 画布宽（像素格数） |
| 2 | height | int | 是 | 画布高 |
| 3 | bg | color | 否 | 背景色，默认透明 |

示例：`["C", 32, 32]` `["C", 64, 64, "#fff"]`

### `pal` - 调色板

```
["pal", [color, color, ...]]
```

定义调色板后，后续所有绘画指令都必须通过 `colorIndex` 显式引用这里的颜色。

示例：`["pal", ["#f00","#0f0","#00f","#ff0","#000","#fff"]]`

### `p` - 单像素

```
["p", x, y, colorIndex]
```

| 位置 | 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| 1 | x | int | 是 | |
| 2 | y | int | 是 | |
| 3 | colorIndex | int | 是 | 调色板索引 |

### `P` - 批量像素

```
["P", [x,y, x,y, x,y, ...], colorIndex]
```

坐标为扁平数组（非嵌套），减少括号 token。

示例：`["P", [0,0, 1,1, 2,2, 3,3], 1]`

### `r` - 矩形

```
["r", x, y, w, h, colorIndex]
["r", x, y, w, h, colorIndex, fill]
```

| 位置 | 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|------|
| 1 | x | int | 是 | | 左上角 x |
| 2 | y | int | 是 | | 左上角 y |
| 3 | w | int | 是 | | 宽 |
| 4 | h | int | 是 | | 高 |
| 5 | colorIndex | int | 是 | | 调色板索引 |
| 6 | fill | 0\|1 | 否 | 1 | 0=描边 1=填充 |

示例：`["r", 0, 0, 10, 10, 0]` `["r", 5, 5, 20, 20, 1, 0]`

### `e` - 椭圆/圆

```
["e", cx, cy, rx, ry, colorIndex]
["e", cx, cy, rx, ry, colorIndex, fill]
```

| 位置 | 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|------|
| 1 | cx | int | 是 | | 中心 x |
| 2 | cy | int | 是 | | 中心 y |
| 3 | rx | int | 是 | | x 半径 |
| 4 | ry | int | 是 | | y 半径 |
| 5 | colorIndex | int | 是 | | 调色板索引 |
| 6 | fill | 0\|1 | 否 | 1 | |

圆 = rx 和 ry 相同：`["e", 16, 16, 8, 8, 1]`

### `l` - 直线

```
["l", x1, y1, x2, y2, colorIndex]
```

示例：`["l", 0, 0, 31, 31, 2]`

### `f` - 填充（flood fill）

```
["f", x, y, colorIndex]
```

### `#` - 注释

```
["#", "说明文字"]
```

用于标记绘制步骤，不影响渲染。

## 颜色格式

调色板内容支持以下格式，优先使用短格式：

| 格式 | 示例 | token 开销 |
|------|------|-----------|
| 3位hex（推荐） | `"#f00"` | ~1 token |
| 6位hex | `"#ff0000"` | ~2 tokens |

绘图指令本身只能使用调色板索引，如 `0`、`1`、`2`。

## 可选参数规则

所有指令遵循统一规则：
1. 参数按位置排列，从左到右
2. 完整项目指令顺序固定为：`C -> pal -> 其余绘图/注释指令`
3. AI 返回的 `actions` 不包含 `C`，但如果存在绘图内容，第 1 条必须是 `pal`
4. 每条绘图指令都必须显式给出 `colorIndex`
5. 仅 `r` 和 `e` 支持尾部可选 `fill`

## 扩展机制

### 添加新指令类型

1. 注册新类型码（1-2 字符，不与已有冲突）
2. 定义参数序列（遵循：必填在前，可选在后，避免与 `colorIndex` 位置冲突）
3. 前端 renderer 添加对应绘制函数
4. 更新 LLM system prompt 的指令表

### 指令选项扩展（未来）

当某类指令需要更多可选配置时，末尾可追加 options 对象：

```jsonc
["r", 0, 0, 10, 10, 1, "#f00", {"opacity": 0.5, "blend": "multiply"}]
```

options 对象只在需要时出现，不影响基础格式的紧凑性。

## Token 对比

绘制一个红色填充矩形 + 蓝色圆形的完整指令：

### 对象格式（旧）：约 45 tokens
```json
[
  {"type":"canvas","width":32,"height":32},
  {"type":"palette","colors":["#ff0000","#0000ff"]},
  {"type":"rect","x":2,"y":2,"w":10,"h":10,"colorIndex":0,"fill":true},
  {"type":"circle","cx":20,"cy":16,"r":6,"colorIndex":1,"fill":true}
]
```

### 数组格式（新）：约 18 tokens
```json
[["C",32,32],["pal",["#f00","#00f"]],["r",2,2,10,10,0],["e",20,16,6,6,1]]
```

### 调色板模式：约 15 tokens
```json
[["C",32,32],["pal",["#f00","#00f"]],["r",2,2,10,10,0],["e",20,16,6,6,1]]
```

复杂场景下（几十条指令），调色板模式 token 节省更显著。

## LLM 图片输入

前端向 LLM 发送的 vision 消息有两类图片来源，共用 OpenAI 兼容的 `image_url` 格式：

```jsonc
{
  "role": "user",
  "content": [
    { "type": "text", "text": "用户的文字描述" },
    { "type": "image_url", "image_url": { "url": "data:image/png;base64,..." } }
  ]
}
```

### 图片来源分类

| 类型 | 触发 | 来源 | prompt 策略 |
|------|------|------|------------|
| 参考图 | 用户上传/粘贴/拖拽 | 用户本地文件 | "参考这张图片，用像素画指令还原/模仿其内容" |
| 画布截图 | 点击"AI审视"按钮 | canvas.toDataURL() | "这是当前画布状态，请分析并根据要求生成修正指令" |

### 参考图输入

- 支持方式：拖拽到对话输入区 / 点击上传按钮 / Ctrl+V 粘贴
- 格式：PNG、JPG、GIF、WebP
- 前端处理：读取为 base64 data URL，压缩到合理尺寸（建议长边不超过 1024px）
- 可同时附带多张参考图
- 输入区显示缩略图预览，可单独删除

### 画布截图

- 点击"AI审视画面"按钮触发
- 自动调用 `canvas.toDataURL("image/png")` 获取当前画布状态
- 截图自动附加到下一条消息，无需用户手动操作
- 可与参考图同时存在（如："参考这张图，看看当前画布还差什么"）

### 混合场景示例

用户上传一张猫的照片，同时附带画布截图：

```jsonc
{
  "role": "user",
  "content": [
    { "type": "text", "text": "参考这张猫的照片，看看我当前画布画得怎么样，帮我补全尾巴部分" },
    { "type": "image_url", "image_url": { "url": "data:image/jpeg;base64,<参考图>" } },
    { "type": "image_url", "image_url": { "url": "data:image/png;base64,<画布截图>" } }
  ]
}
```

System prompt 中需告知 LLM 如何区分两类图片（按顺序约定或在 text 中标注）。

## LLM 输出约定

要求 LLM 返回时：
1. 只输出 JSON 数组，不加任何解释文字
2. 如果必须包含文字，用 markdown code block 包裹 JSON
3. 增量模式：返回新增指令（追加到已有列表）
4. 全量模式：返回完整指令列表（替换）
5. 通过对话上下文区分增量/全量
