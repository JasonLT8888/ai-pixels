# 功能状态与缺口清单

本文档用于回答三个问题：

1. 现在已经能做什么
2. 哪些能力在旧文档或产品范围中提到过，但还没实现
3. 哪些功能值得作为后续 backlog

## 1. 已实现能力

### 核心闭环

- 通过自然语言生成像素画指令
- 通过 `talk + actions` 结构化返回结果
- 画布实时预览和步骤回放
- 手动应用 AI 生成的预览到项目指令

### 会话系统

- 一个项目下可创建多个 chat
- 每个 chat 有独立画布尺寸
- 每个 chat 保存自己的消息历史
- 支持清空消息、删除 chat、清空整个项目的 chats
- 支持对长对话做压缩摘要
- SSE 中断后支持 retry，避免重复插入上一条 user 消息

### 图像输入

- 上传图片
- 拖拽图片
- 粘贴图片
- 自动压缩过大的输入图片
- 将 assistant 输出中的图片再次送入输入框
- AI 自行检查当前渲染图

### 指令编辑与播放

- 统一规范化项目指令
- JSON 粘贴编辑
- JSON 导出带画布尺寸的 actions bundle
- 拖入 JSON 文件或选择文件后自动填充编辑框
- 前置指令与绘制步骤分开展示
- 自动播放、手动前进后退、步速控制
- comment 指令在画布上显示为说明提示

### 工作区操作

- 顶部文件菜单支持导入 JSON、导出 JSON、导出 PNG

### 配置系统

- 多个 LLM 配置档案
- 选择默认配置
- 拉取上游模型列表
- 单独设置 `context_window`
- 单独设置 `compress_threshold`
- 追加提示词存储

## 2. 已确认缺失的功能

以下能力在 README、旧架构文档或阶段规划中已经出现，但当前代码未实现。

### 高优先级

- 项目导出 JSON
  - 现状：README 旧版本提过，但没有对应接口或下载入口
  - 影响：项目迁移与备份不方便

### 已补齐但仍可增强

- 项目导出 PNG
  - 现状：前端已支持将当前播放到的画面直接导出为 PNG，本地浏览器完成下载
  - 仍可增强：自定义文件名、导出最终完整画面、批量尺寸导出

### 中优先级

- 左侧绘图工具栏
  - 现状：App 左侧仍是占位内容“工具栏”
  - 影响：目前只能通过 AI 或 JSON 编辑绘图

- 画布交互式绘制
  - 现状：画布只支持缩放和平移，不支持点击/拖拽绘图
  - 影响：缺少可视化编辑闭环

- 指令列表级别的增删改排序 UI
  - 现状：InstructionPanel 只读，真正编辑依赖 JSON 文本框
  - 影响：对非技术用户不友好

## 3. 建议补充的 backlog

以下不一定是原始需求，但从当前产品演进上看具有实际价值：

- 项目复制 / fork
- 项目导入 JSON
- 单条消息重发与编辑
- chat 搜索 / 筛选
- 指令差异对比
- 导出会话快照
- 记录模型耗时与响应元信息
- 记住每个项目或 chat 的缩放与视口位置
- 更明确的离线模式提示

## 4. 不要误判为已实现的内容

以下能力容易因为旧文档措辞而被误解为已完成，当前应视为未实现：

- `POST /api/llm/chat-vision`
- `GET /api/llm/models`
- `GET /api/projects/:id/export/json`
- 绘图工具栏与鼠标绘图

## 5. 建议的后续实现顺序

1. 导出 JSON
2. 左侧工具栏最小版本
3. 画布交互式绘制
4. 指令列表可视化编辑

这个顺序的原因是：先补齐项目迁移能力，再逐步提升直接编辑体验。
