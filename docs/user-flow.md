# 用户使用流程

本文档描述当前项目里真实可用的用户流程，重点是帮助后续使用者和 AI 理解“用户实际上怎么用这个产品”。

## 1. 首次启动

1. 安装依赖并启动前后端
2. 打开页面后，系统会尝试加载已有项目
3. 若本地数据库中还没有项目，前端会自动创建一个默认项目

## 2. 首次配置 LLM

1. 点击右上角设置按钮
2. 新增或编辑一个 API 配置
3. 填写 API 地址、API Token、模型名称
4. 可选：点击“拉取模型列表”确认上游可用模型
5. 可选：设置上下文窗口和自动压缩阈值
6. 将该配置设为默认并保存

说明：

- 内置系统提示词只读，不可直接修改
- 用户只能通过“追加提示词”补充额外约束

## 3. 创建或进入项目

当前前端启动后会优先打开第一个项目，若没有项目则自动创建默认项目。

项目内保存的是：

- 项目名称
- 画布基础尺寸
- 当前完整指令集

## 4. 创建 chat 并设定画布

1. 在右侧对话区域创建新 chat
2. 输入 chat 标题与画布尺寸
3. 进入该 chat 后开始对话

关键点：

- chat 有自己的画布尺寸
- AI 不允许直接输出 `canvas/C`
- 系统会按当前 chat 画布自动补全项目完整指令中的 canvas

## 5. 纯文字生成像素画

1. 在输入框输入自然语言描述
2. 发送消息后，前端通过 SSE 接收 AI 流式回复
3. 对话区会先显示文字说明，再逐步出现结构化预览
4. 若 AI 返回了有效 `actions`，前端会生成过程回放和最终预览
5. 用户确认后，将预览推送到项目指令

当前推荐的使用方式：

- 先让 AI 描述构图与步骤
- 确认预览接近目标后再应用到项目

## 6. 使用参考图

当前支持三种方式添加图片：

- 上传文件
- 拖拽文件到输入区域
- 直接粘贴图片

流程如下：

1. 将图片加入输入区
2. 输入文字要求，如“参考这张图生成像素画”
3. 发送后，后端会把文字和图片一起发给上游 LLM
4. AI 返回 `talk + actions`
5. 用户查看预览并决定是否应用

补充：

- 图片过大时，前端会先缩放压缩
- 图片会以 JSON 字符串数组形式存到消息记录中

## 7. 自行检查当前画面

这个流程用于“让 AI 看看它刚画出来的结果是否有问题”。

1. 前端将当前项目指令渲染成 PNG data URL
2. 系统整理最近相关聊天上下文
3. 将截图和上下文一并发给同一个聊天接口
4. AI 重新分析当前画面并返回修正建议或修正动作
5. 用户查看新预览后决定是否应用

这个流程适合：

- 检查构图是否跑偏
- 检查颜色是否不对
- 检查局部细节是否需要补充

## 8. 查看与回放指令

项目中央区域提供两种主要查看方式：

- 指令列表
- 播放器

用户可以：

1. 逐步前进或后退
2. 自动播放绘制步骤
3. 调整播放速度
4. 查看 comment 指令带来的阶段说明

注意：

- `canvas` 和 `palette` 属于前置指令
- 它们会参与实际渲染，但不会计入主要步骤数

## 9. 手动编辑 JSON

当 AI 结果接近目标但还需要精修时，可以直接编辑 JSON：

1. 在 JSON 编辑区查看当前完整项目指令
2. 手动修改或粘贴新的指令数组
3. 点击应用
4. 前端和后端都会做规范化与保存

适合的场景：

- 修一个坐标
- 改一个颜色索引
- 追加几条简单指令

## 10. 管理 chat 历史

当前可以进行的 chat 操作包括：

- 创建 chat
- 切换 chat
- 删除 chat
- 清空单个 chat 消息
- 清空整个项目下所有 chats
- 压缩当前 chat 的旧消息

对话很长时，建议使用压缩功能，减少上下文负担。

## 11. 长对话压缩

压缩的大致逻辑是：

1. 保留最近若干轮 user/assistant 消息
2. 把更早的消息发给 LLM 生成摘要
3. 保存摘要到 chat 记录
4. 后续请求时注入摘要，而不是反复注入所有旧消息

这样可以：

- 降低上下文消耗
- 保留主要语义连续性
- 避免 chat 越聊越慢

## 12. 失败与重试

当流式请求中断且没有收到完整结果时：

1. 前端会把这次请求视为失败
2. 用户可以点击重试
3. 重试时会带上 `retry_last_user`
4. 服务端若发现最后一条 user 消息与当前重试内容一致，就不会重复写入数据库

这保证了：

- 用户不需要手动复制上一条消息
- 会话记录不会因为重试而出现重复 user 消息

## 13. 当前使用边界

以下能力目前还不在真实用户流程中：

- 直接导出 PNG
- 直接导出 JSON
- 用鼠标在画布上直接绘图
- 在可视化列表里逐条编辑指令

这些内容见 [feature-status.md](feature-status.md) 和根目录 [待办.md](../待办.md)。
